상속(inheritance)

	1. 기존에 선언된 클래스의 필드를 다른 클래스에서 사용하고자 할 때
	   상속을 한다.

	2. 여러 클래스를 만들고자 할 때 공통요소들을 묶어서 부모 클래스를
	   먼저 만든 후 상속해준다.

class A{
	A필드
}

class B extends A{
	A, B필드
}

A : 부모 클래스, 상위 클래스, 슈퍼 클래스, 기반 클래스
B : 자식 클래스, 하위 클래스, 서브 클래스, 파생 클래스

생성자
	자식 객체는 자식 생성자를 호출하여 객체화를 한다.
	메모리에 자식 필드만 올라갔다면 자식 객체로 절대 부모 필드를
	접근할 수 없다. 따라서 자식 생성자를 호출하게 되면
	부모 생성자가 자동으로 호출되고 따라서 자식 객체는
	부모 필드도 접근할 수 있게 된다.


다형성(Polymorphism)

	2. 오버라이딩(Overriding) : 재정의
		부모 필드에 이미 a라는 메소드가 존재한다면
		부모 생성자가 먼저 호출되기 때문에 a()가 메모리에 먼저
		올라간다.
		그 다음 자식 생성자 호출시 같은 이름의 메소드가 선언되어
		있다면, 같은 이름으로 2개가 만들어지지 않고 기존에 만들어진
		부모 a()메소드에 자식에서 작성한 코드의 주소값이
		덮어 씌워진다. 그러므로 자식 객체로 a()를 사용하면 
		재정의된 기능으로 사용되며, 이것을 재정의라고 한다.

up casting
	부모 타입의 객체에 자식 생성자를 넣는 기법

down casting
	자식 타입의 객체에 부모타입의 객체를 넣는 기법
